## 1. はじめに（背景・動機）

『バイオハザード』『サイレントヒル』『クロックタワー』など、1990年代後半のサバイバルホラーゲームは、**固定視点（プリレンダ背景に固定カメラ）** という演出によって独特の恐怖を作り上げてきた。プレイヤーは自分のキャラクターを第三者の画角で見続け、画面外から迫る脅威・カメラ切替の瞬間に現れる敵・自分の死角に立つ何者かといった、一人称視点では成立しない恐怖を体験した。

しかし2000年代以降、ゲームは三人称追従カメラ・一人称視点へと収斂し、固定視点演出は技術的・操作性の理由から廃れていった。VRの台頭はこの流れをさらに推し進め、現代のVRホラーは「自分の目で直接見る」一人称没入を前提としている。

本作は、この潮流に逆行し、固定視点演出をVRと実空間の組み合わせによって再構築することを提案する。体験者はHMDを装着して実空間を自身の足で移動するが、HMDに提示されるのは空間内に設置された複数の固定カメラから取得した自身を含む映像のみである。すなわち、視覚は他者の画角に固定されたまま、移動と姿勢制御のみが体験者自身の身体に委ねられる。この視覚と身体運動の分離は、固定カメラの画角外に対する不可知性、第三者視点に常時晒されることによる被観測感、ゾーン間のカメラ切替に伴う方向感覚の不連続性といった、一人称VRおよび追従カメラ型ゲームのいずれにおいても生起しない知覚条件を作り出す。

## 2. 体験コンセプト

> 体験者が「何を体験するのか」を1〜2文で言い切る。
> 例: 「固定視点カメラ越しにしか見えない世界をVRで歩き、視点の制約から生まれる恐怖と発見を味わう体験」

### 2.1 コンセプト一文
**固定視点カメラ越しにしか見えない実世界を、自分の体で歩いて探索するホラー体験。**

### 2.2 体験のターゲット感情 / 主訴求
- **見られている恐怖**: 自分自身が常に第三者の画角に映っている不安
- **死角への恐怖**: 視界外（自分の背後・カメラ画角外）に何があるのか分からない
- **身体と視点の乖離**: 自分の足で歩いているのに、視点だけが他人の目線
- **CG合成による境界の揺らぎ**: 実世界の映像にホラー演出が重なり、何が現実で何がフィクションか分からなくなる

### 2.3 想定する体験者
- VRに触れた経験のある10代〜40代
- 90年代サバイバルホラーゲーム世代には強くノスタルジーを刺激し、若年層には「画面の中の主人公の感覚を自分で体験する」新規体験として届ける

## 3. 体験内容（シナリオ / フロー）

体験スペース中央には腰高程度の **仕切り壁** を配置し、空間を3つのゾーンに分割する。各ゾーンに固定カメラを1台ずつ設置し、体験者の現在位置に応じて自動的にカメラ映像が切り替わる。

### 3.1 全体フロー（約3分）

1. **導入（〜30秒）**: HMD装着、キャリブレーション。最初は通常のパススルー映像で立ち位置を確認。
2. **シーン1: 廊下（〜45秒, カメラ①, 立位移動）**
   斜め上からの俯瞰カメラ。体験者は仕切り壁に沿って歩き、奥へ進む。映像にはノイズ・走査線フィルタが乗り、画面端に「何か」が一瞬映る演出。
3. **シーン2: 角を曲がる（〜45秒, カメラ②, 立位移動）**
   カメラ位置が切り替わり、視点が反転する瞬間に方向感覚が一度失われる。仕切り壁の裏側を覗き込む / 屈むなどの動作で映像内に CG オブジェクト（落書き・血痕・人影）が出現。
4. **シーン3: 部屋（〜45秒, カメラ③, 着座）**
   椅子に座らされ、正面の固定カメラに自分が大きく映る。背後（実空間にも CG にも）に異変が現れ、体験者は振り返るが、カメラは動かないため自分の後頭部しか見えない。
5. **終了（〜15秒）**: 暗転、HMD取り外し。

### 3.2 インタラクションの肝

- **位置トラッキングによるカメラ切替**: 体験者が物理的にどのゾーンにいるかで映像が切り替わるため、「移動 = カメラ切替」という固定視点ゲーム特有の体験が、自分の身体で再現される。
- **CG合成の能動性**: 体験者がしゃがむ・特定の位置に立つ・後ろを向くといった行動に応じて、映像内のCG演出が変化する。

## 4. 技術構成

### 4.1 システム全体図

体験スペースに設置した複数のスマートフォン（固定カメラ役）が、Wi-Fi経由でMJPEGストリームを配信。Quest 3 上の Unity アプリがこれを受信・デコードし、HMD内に「眼前のスクリーン」として表示する。体験者の頭部位置は Quest 3 のインサイドアウトトラッキングで取得し、ゾーン判定によってアクティブなカメラ映像を切り替える。

> 図1: システム構成図 — TODO（描画予定）

### 4.2 ハードウェア
- HMD: Meta Quest 3（インサイドアウト6DoFトラッキング）
- 映像入力: スマートフォン3台（DroidCam / IP Webcam, MJPEG over Wi-Fi）
- ネットワーク: 体験会場内のローカル無線LAN（外部ネット非依存）
- 設置物: 仕切り壁（高さ約1m）、椅子1脚、カメラ三脚3本

### 4.3 ソフトウェア
- Unity 2022.3 LTS / URP（Android / IL2CPP / ARM64）
- Meta XR All-in-One SDK + OpenXR Plugin + XR Interaction Toolkit
- 映像取り込み: MJPEGストリームをUnity内でテクスチャに非同期デコード（GCアロケーション抑制）
- 映像加工: Shader Graph によるリアルタイムフィルタ（走査線・色収差・ノイズ・ビネット）
- CG合成: ストリーム上に3Dオブジェクトを重畳（Quad/Plane + αブレンド）

### 4.4 技術的チャレンジ / 新規性

**(1) 固定視点演出の現実空間への移植**: 90年代サバイバルホラーの固定カメラ演出は画面の中の出来事だったが、本作はこれを実空間と体験者の身体に持ち込む。VR文脈では一人称没入が主流だが、あえて第三者視点を強制することで、既存VR体験にない不安・恐怖の質を生む。

**(2) 90Hz維持下での多重MJPEGデコード**: 複数スマホからのMJPEGストリームを同時受信しつつ、Quest 3 で90Hz表示を維持する。フレーム内のGCアロケーションを排除する非同期デコードパイプラインを構築する。

**(3) 位置駆動の映像演出**: 体験者の物理位置・姿勢をトリガーに、映像のCG合成・カメラ切替・フィルタ強度を動的制御する。「移動するとカメラが切り替わる」という固定視点ゲームの構造を、画面入力ではなく身体運動で発火させる点が新規。

## 5. 演出 / アート方針

### 5.1 ビジュアル
- **ベース**: 90年代サバイバルホラーゲームの監視カメラ風UI（走査線・タイムスタンプ・カメラ番号表示）
- **フィルタ**: 低解像度化、色収差、ビネット、フィルムグレイン、ホワイトノイズ
- **CG合成**: 実映像に対し、血痕・落書き・影・歪んだ人影を α合成。実空間には何もないのに、HMD越しの映像にだけ存在する「見えるが触れないもの」を作る
- **カメラ切替演出**: 切替時に一瞬の暗転 + ノイズ。固定視点ゲームのロード演出を踏襲

### 5.2 サウンド
- **環境音**: 通気口・配線のハム音・遠くの足音など、空間を実在感のある「どこかの建物」に変える低音域
- **空間音響**: Quest 3 の空間オーディオで、画面外（=固定カメラの画角外）から音を鳴らす。視覚で確認できないため恐怖が増幅される
- **インタラクション音**: 体験者の移動・着座・振り向きに応じてSEが発火

### 5.3 空間演出
- 物理仕切り壁にも軽い装飾（暗幕・小物）を施し、HMDを外しても世界観が崩れないようにする
- 体験中は会場照明を落とし、外部視覚情報を遮断

## 9. むすび

本作は、VR体験における視点設計の主流である「一人称没入」に対し、あえて「自分自身を第三者の固定画角で見続ける」という逆方向の設計を提示する。固定視点カメラというかつてのゲーム表現を、画面の中ではなく実空間と体験者の身体に差し戻すことで、現代のVRでは生まれにくい種類の恐怖・不安・身体性の揺らぎを生み出すことを目指す。
さらに本企画は、単なるホラー作品にとどまらず、「視覚と身体の乖離が体験者に何をもたらすか」という、VR研究にとっても示唆ある問いを内包する。実装と展示を通じて、視点設計の選択肢を一つ拡張することに貢献したい。
