## 1. コンセプト一行（ビデオ全体を貫く軸）

> **「あなたは——空間に据えられた固定カメラ越しにしか、自分を見られない。」**

90 年代サバイバルホラーの固定視点カメラを、VR と実空間で“画面の外”に解き放つ。
体験者の視点は**空間に固定されたカメラ**に縛られ、自分の足で実空間を歩いても、見えるのは“固定カメラに映った自分”だけ。
（※「他人＝誰かの視点」ではなく、**空間に置かれた固定カメラの視点**。視点は動かず、動くのは画面の中の自分。）

## 2. 2 分の骨格（合計 1:55 + 予備 5s）

```
[0:00–0:05]  コールドオープン（黒地・CCTV タイムスタンプ・テープノイズ）
─────────── A. 作品の概要（約 55s）── 審査軸:「体験の特徴・面白さ」 ───────────
[0:05–0:18]  90年代固定視点ホラーへのオマージュ + タイトル提示
[0:18–0:32]  ひねり:「その固定カメラの中に、あなた自身が立っていたら？」
[0:32–0:50]  体験の核（視界外の恐怖 / 被観測 / 視点と身体の乖離）= 予告イメージ映像
[0:46–0:54]  主要演出を予告（見ている映像の一部が別撮り映像にすり替わり、居ないものが映る）
─────────── B. 作品の特徴・原理・プロトタイプ（約 55s）── 審査軸:「チャレンジの魅力 + 実現の説得力」 ──
[0:58–1:08]  淡々とした導入「この体験は、実際に動くプロトタイプです」→ 実機現実カット（三脚スマホ + Quest3 装着者）
[1:08–1:30]  ★決め画: 分割画面（実空間の歩行 ←→ HMD 内 CCTV 映像が位置で自動切替）
[1:30–1:42]  原理 3 ステップ図 + レイテンシ < 100ms
[1:42–1:53]  映像加工 FX ＋ **主要演出＝映像の一部を別撮り映像に差し替え**（ScreenOverlayController/OverlayCue。実装済の仕掛け・見せる例はイメージ）
[1:53–2:00]  エンドカード（タイトル・キーワード・「VR 学会で展示」）
```

詳細カット割りは [storyboard.md](storyboard.md)。

## 3. なぜこの構成か（審査軸への当て方）

- **A（概要）= 面白さ**: 既知のもの（90 年代固定カメラホラー）を一行で想起させ、「中に自分が立つ」というひねりで“具体化された体験の特徴”を一撃で伝える。言葉でなく**画と空気**で売る。未完成部分は予告イメージで“こうなる”を提示。
- **B（特徴）= チャレンジ + 説得力**: **冒頭で「すでに動く」を見せる**のが最重要。本企画の技術的新規性＝「身体運動で固定カメラ構造が切り替わる」が、分割画面の実機映像で**証明**される。机上の企画でなく動くプロトタイプがある＝実現の説得力。残りの CG 演出は“あと何を足すか”として誠実に提示。

## 4. 撮れる素材（実装の現状ベース）

実機で動作確認済み = **撮れる**:
- スマホ固定カメラ → Quest3 への MJPEG ライブ表示（Phase 2）
- 複数カメラ手動切替（A/B ボタン, Phase 2.5）
- **位置連動の自動カメラ切替**（ゾーン移動でカメラが変わる, Phase 2.7）← 決め画
- HMD 内ゾーン校正（演出として一瞬使える）
- 映像加工 FX 4 系統（CRT 走査線 / 色収差 / 埃 / Sobel）= Editor で見栄えカット可（Phase 3、実機 FPS は要確認）

未着手 = **予告イメージ映像 / AI 生成で補う**:
- スクリーン外 3D 演出・CG 合成（血痕・人影・落書きを実空間映像に重ねる, Phase 4）

## 7. 撮影・編集の品質ルール（ホラーとして効かせる）

- **誇張しない（審査で最重要）**: 遅延の数値は断定しない（自プロジェクト実測 100–200ms・企画書も「目標」なので `<100ms` 断定は誇張）。出すなら実測カット（ストップウォッチ写し込み）か定性表現。90Hz/fps も実機計測なしに断定しない。
- **「実機」と「イメージ映像」を画で分ける**: 実機カット=テロップ「実機プロトタイプ」、未実装の予告=「イメージ映像」。実機は CCTV ノイズ質感、予告は明確に CG 調にして視覚で区別。これを怠ると“できてないのにできてるフリ”と見抜かれ B の説得力が崩壊する。
- **決め画はワンショットで証拠化**: 歩く人と HMD 映像を 1 フレームに同居（左右別撮りの分割はヤラセに見える）。
- **手ブレ厳禁の実機画**: 外部カメラは三脚 or ジンバル。CCTV パートは“動かない画角”が命。
- **暗所 + 1 光源**: 実空間は暗めに。スマホカメラの露出はロック（人が通っても画が動かない）。
- **CCTV 質感を編集でも足す**: カメラ番号 `CAM 01`・タイムスタンプ・走査線・軽い色落ち・四隅ビネット。実機 FX とダブらせず、足りない分を編集で。
- **間（ま）で怖がらせる**: カットを詰めすぎない。静止 → 一拍 → 異変、の呼吸。
- **音が 7 割**: 低い持続音 + カメラ切替のノイズ/クリックで“監視されている感”を作る。

## 8. 審査員レビュー（2026-06-16・サブエージェント 4 観点）と反映

ビデオ計画を IVRC 審査員役 4 体（体験/技術/映像尺/新規性）で批判的レビュー。判定は A 通過圏・B 通過圏(条件付)・C 良(尺要修正)・D ボーダー。**反映済みの HIGH 修正**:

1. **遅延<100ms / 90Hz の誇張を削除** → 断定をやめ実測カット/定性表現へ（narration「遅延の見せ方」, figure-1, 本書 §7）。
2. **実機/イメージ のテロップを台本に実体化** → storyboard・narration の各カットに明記。
3. **尺の正規化** → ⑩の破綻・⑧早口・B1 22→16s・A の余りを再配分し 1:58 に収めた（narration v2）。
4. **A の恐怖核を実機実写化** → A3「視点（固定カメラ）は動かず、画面の中の自分だけが振り返る＝視点と身体の乖離」を実写必須に（HMD-3 を★化）。
5. **B1 をワンショット証拠化＋フォールバック** → 現実と HMD 映像を 1 画角に同居（capture-shotlist）。
6. **新規性「操作ゼロ・身体運動駆動」を明示** → figure-1・B2 テロップ「切替＝身体の移動だけ」（ナレで誇示しない）。
7. **トーンの謙虚化（審査員レビュー 2026-06-17）** → ⑥の否定形対比「構想ではありません／すでに動いています」を削除、⑦の売り文句「操作は、いりません」を削除、⑩「住まわせていく」→「映し込んでいく」、テロップ「無編集」削除（ヤラセ疑い前提に見えるため）。
8. **実機/イメージ表記の是正** → 動画で使う映像はほぼ全て実機（プロトタイプ）。真の「イメージ映像」は未実装 CG の A5・B4 のみ。よって ⑥ を「ここから実機」式の言い方から「この体験は、実際に動くプロトタイプです」へ（A も実機なので途中から実機とは言わない）。COLD/A1（無人室内の実機映像）の「イメージ映像」バッジを除去。

**ユーザー判断に残す（反映保留）**: 一人称 VR との対比カット B1.5 を入れるか（尺と素材次第・storyboard 参照）／体験者の身体リアクションを匿名で撮るか（capture-shotlist）／学術主張（企画書 [3] Slater 引用）はビデオで前面に出さない方が安全。
