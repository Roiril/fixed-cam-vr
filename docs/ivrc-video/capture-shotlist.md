# 撮影 / 録画リストと段取り

ビデオに必要な映像素材を「どこで・どう撮るか」で 4 系統に分ける。**★= 最優先（これが無いと成立しない）**。
撮影自体はユーザー作業（クロードには不可）。Editor 録画と作図はクロードが代行可。

---

## 系統 1: 実機・外部カメラ実写（ユーザー）

別のスマホ/カメラで、体験している現実の様子を撮る。三脚必須（手ブレは禁物）。

| ID | ★ | 内容 | カット | コツ |
|---|---|---|---|---|
| EXT-1 | ★ | 三脚スマホ 3 台 + Quest3 装着者が立つ部屋の**引きの全景** | B0 | スマホ 3 台が「カメラ源」と分かる画角。少し暗めの部屋 |
| EXT-2 | ★ | **ワンショット証拠**: 1 フレーム内に〈中央→右→左と歩く装着者〉＋〈HMD 内映像を映した PC モニタ/キャスト画面〉を**同時に**収める。境界の養生テープも入れる | B1 | これが B の心臓。**左右別撮りの分割はヤラセに見える**ので、現実と HMD 映像を 1 画角に同居させて「同じ瞬間に歩行と切替が起きる」を編集なしで証明。2〜3 テイク |
| EXT-3 | △ | 装着者がしゃがむ/振り返る所作のアップ | A3 | 「身体は動くのに視点は動かない」を外から見せる補強 |
| EXT-4 | △ | スマホを三脚にセットする手元・起動画面（メイキング的） | B2 補助 | 原理図の繋ぎに使える。無くても可 |

## 系統 2: HMD 内録画（ユーザー）

体験者が実際に見ている CCTV 映像。**Quest 3 の録画機能**（クイック設定 → 録画、または PC へキャスト録画）。

| ID | ★ | 内容 | カット | コツ |
|---|---|---|---|---|
| HMD-1 | ★ | 無人の固定カメラ映像（薄暗い部屋・誰もいない不穏な画） | 0:00 / A1 | 体験開始前に数十秒録っておく |
| HMD-2 | ★ | **歩行中にゾーン跨ぎでカメラが自動切替**する瞬間（B1 の同時記録に使う） | B1 | 各ゾーンで 2〜3 秒静止してから移動。1 テイクで切替を 2 回起こす |
| HMD-3 | ★ | **視点と身体の乖離の決め画（実写必須・AI不可）**: 体験者が振り返る → スクリーン（固定カメラ映像）の中の自分も振り返る。だが視点＝カメラは動かず画角外は見えない。死角の暗がりに何かが一瞬よぎる | A3 | 審査員が「実際に不気味」と感じる心臓部。**“視点は固定カメラのまま、動くのは画面の中の自分”**が伝わる構図で。座って正面カメラに自分が大写し→背後に異変、でも自分は画面の中、でも可 |
| HMD-5 | ○ | 撮影スマホ画面と HMD 内映像に**同じストップウォッチ**を写し、ズレ（遅延）を 1〜2 秒 | B2 補助 | 遅延を“口でなく実測の画”で示す。数値を出すならこの実測を根拠に（[narration.md](narration.md) 遅延の見せ方） |
| HMD-4 | ○ | A/B ボタンでの手動カメラ切替（切替の滑らかさを見せる） | 補助 | 自動切替が主役なので短くて可 |

> Quest 録画の注意: 録画は既定でガーディアン/UI が映り込むことがある。**録画中に余計なシステム UI を出さない**。解像度は最高設定。録画した mp4 は `adb pull` か Meta アプリ経由で PC へ。

### HMD 内映像をクリーンに録る代替: web-compositor 録画

HMD 録画にノイズ（UI 映り込み等）が乗る場合、**配信フレームそのもの**を PC でクリーンに録れる:

1. スマホで fixed-cam-streamer 起動 → `tools/web-compositor/serve.ps1` で起動 → ブラウザで `http://localhost:8099/`
2. 「⏺ 配信を録画」で MJPEG を webm 録画（`tools/web-compositor/captures/` に保存）
3. これは“カメラ単体の生映像”なので、自動切替の演出は付かない点に注意（切替は HMD-2 が要る）。CCTV の**素材クリップ**や FX 合成の確認用に最適。

詳細: [.claude/memory/web_compositor.md](../../.claude/memory/web_compositor.md)

## 系統 3: Unity Editor 画面録画 / 静止画（クロード代行可）

実機不要。見栄えカットを Editor で作る。**「FxSandbox を出して FX を ON にした画」**を録る。

| ID | 内容 | カット | 作り方 |
|---|---|---|---|
| ED-1 | FX モンタージュ（CRT 走査線 / 色収差 / 埃 / Sobel エッジ）を素映像にかける比較 | B3 | `Tools > FixedCamVr > Setup > Setup FxSandbox Scene` → 各 Phase の GameObject を SetActive 切替して Game ビュー録画 |
| ED-2 | ゾーン切替のギズモ可視化（俯瞰でゾーン Box と HMD 位置マーカーが動く） | B1/B2 補助 | PlayerZone Sandbox を Scene ビュー俯瞰で。位置駆動の仕組みを抽象的に見せる |
| ED-3 | スクリーン枠 + CCTV UI（カメラ番号/タイムスタンプ）の見本 | A 全般 | Main シーンの MjpegScreen 表示を録る |

> クロードに「FxSandbox の FX サンプルを Editor で撮って」と言えば、Unity MCP で FxSandbox 生成→各 FX を切替→Game ビュー or スクショ取得まで実行する（共有 Editor なので TableDuo 作業と競合しないタイミングで逐次）。

## 撮影当日の段取り（半日想定）

1. **部屋設営**: 4 畳半以上、暗め、1 光源。中央に仕切り壁（腰高）。床に養生テープでゾーン境界（中央/右/左）。
2. **スマホ 3 台を三脚に固定** → fixed-cam-streamer 起動 → レンズ選択・**露出ロック**（人が通っても画が動かないように）→ 各 IP を `Phone01/02/03.asset` に転記。
3. **Quest3 を Main シーンのビルドで起動** → `Setup Main Demo Scene` 済みの APK。ゾーン校正（両グリップ長押し）で部屋に合わせる。
4. **動線リハ**: 中央→右→左の歩行を 1 回。HUD で ZONE 行が切り替わるのを確認。
5. **本番録画**: EXT-2（外部カメラ）と HMD-2（HMD 録画）を**同時スタート**。同じ歩行を 2〜3 テイク。各ゾーンで 2〜3 秒静止してから移動（編集で切替が見やすい）。
6. EXT-1 全景、HMD-1 無人画、HMD-3 振り返り を追加で押さえる。
7. 撮れ高チェック → 足りないカットを Editor(系統3)/AI(系統4) で補完。

## 間に合わない時のフォールバック（審査員D「未撮影依存が高い」への保険）

決め画 B1（ワンショット証拠）が撮れないと B の説得力が崩れる。代替を用意しておく:

- **B1 代替**: Unity Editor で OVRHeadsetEmulator / キーボードでゾーン移動を再現 → Game ビューでカメラ自動切替を録画。テロップ「動作原理の再現」と明示して使う（実機が本命・これは保険）。
- **A 恐怖カット代替**: A3 の被観測（後頭部しか映らない）は HMD 録さえ撮れれば成立。部屋設営が間に合わなくても、狭い場所＋スマホ 1〜2 台で最低限の A2/A3 は撮れる。
- 撮影は **B1（決め画）→ A3（被観測）→ B0 全景** の順で優先確保。残りは Editor / AI で補完。

## 体験者のリアクションと匿名性（トレードオフ・要判断）

審査員指摘: 体験者が「びくっとする／後ずさる」反応が一瞬映ると訴求が跳ねる（恐怖の社会的証明）。一方でダブルブラインドだと顔・特定情報は出せない。

- **両立案**: 顔は映さず、**身体の反応**（肩がすくむ・一歩下がる・手が止まる）を後ろ姿/横位置で撮る。これなら匿名のまま“怖がっている”が伝わる。
- ビデオもダブルブラインド要求かは要確認（README タスク①）。不明なら顔は出さない前提で撮る。

- [ ] HMD 録画にシステム UI / ガーディアン枠が映り込んでいない
