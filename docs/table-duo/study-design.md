# 手だけアバターとの対人インタラクション調査 — 設計書

作成: 2026-06-11 / 監査済み（1ペアのパイロット後にプロトコル凍結）
実施プロトコル → [study-protocol.md](study-protocol.md) / 実装計画 → `.claude/plans/2026-06-11_table-duo_study.md`

## 1. 目的

**「手だけの存在」と人はどうコミュニケーションするか**を、TableDuo（テーブル2人VR）上で半構造化観察する。
手は喋れない（ジェスチャーのみ）。人間アバターは喋れる（生声）。手話を知らない操作者同士。

### リサーチクエスチョン

| # | 問い | 主な計測 |
|---|---|---|
| RQ1 | 手話を知らない人は**片手**でどれだけ意思疎通できるか | 質問ゲーム成績（紙）+ ジェスチャー発明の観察 + 運動量ログ |
| RQ2 | カードを手に「見せる」とき、**手のどこへ向けるか**（掌/指先/手首…） | カード法線×手ランドマーク角度の分布（自動ログ） |
| RQ3 | 人は手の**「目」がどこにあると思うか** | ①頭方向プロキシ ②カード照準（RQ2と共用） ③**図指差し（必須・主計測）** + 操作者側の自己モデルも聴取 |
| RQ4 | **発話の非対称**（人=声 / 手=無言）は対話をどう形作るか | Yes/No 質問への収束、発話の構造変化（室内録音の事後注釈） |

## 2. 体験構成（条件）

### 基本条件（デフォルト）

- 手役アバター: **右手のみ**（左手は両者に非表示。操作者自身のローカル描画も右手のみ＝身体感の一貫性）
- **頭マーカー無し**。「どこに話しかければいいか分からない」曖昧さ自体が RQ2/RQ3 の現象
- 手役は発話禁止（ささやき・口音も禁止）。笑い等の漏れは**失敗ではなくデータ**（崩壊瞬間として記録）
- 人役: 従来のフルアバター（頭＋胴体＋両手）、生声で自由に発話
- ジェスチャー語彙は**教えない**（即興こそデータ）。禁止事項のみ伝える

### 対照・バリアント条件（ファシリテータ判断で投入）

| 条件 | 用途 |
|---|---|
| 頭マーカー ON | 「目の位置」誘導の対照 |
| 空書（空中に文字を書く）禁止ラウンド | より深い即興を引き出す難化 |
| 配置課題に「向き」追加 | 協調課題の難化（位置のみ→位置+向き） |
| 両手モード | 帯域比較（後日。基本セッションでは使わない） |

### 役割と起動

- 役割は host/client と独立に `tdv_role full|hand` で起動時指定（adb extras）
- **入れ替え時は HMD を交換しない**（衛生・IPD）。各自の Quest のまま adb で役割を逆にして再起動（~1分）
- 空間アライン不要（席=各自のトラッキング原点。再起動後も自動）

## 3. セッション構成（概要・詳細は protocol）

40分/ペア目安。順序: 自由対面 → カード課題 → 質問ゲーム → 協調配置 → **役割交代** → 短縮再実施 → 事後インタビュー。
開始時に**同期クラップ**（音+映像+モーションスパイク）で全記録ストリームの時刻を揃える。

## 4. 計測・記録

### 自動ログ（SessionLogger・ホスト側）

ホストは全員の pose を受信済みなのでホスト1箇所で記録する。**起動ごとにタイムスタンプ付きファイル名**（役割交代の再起動で上書きしない）。

- 30Hz 行: epoch_ms / 両者の頭 pose / 両者の手首 pose / **手役の7ランドマーク**（手首・掌中心・5指先。bone 回転から算出）/ pinch / tracked
- イベント行: grab / release（対象 id）/ トラッキングロスト・復帰 / フェーズマーク
- カード保持中: カード pose（法線方向の照準分析用）
- フェーズマーク: ファシリテータが PC から `curl http://<host>:7780/mark?label=phase2` （FacilitatorMarkServer）

### 既存の録画（併用）

- 実機の AvatarPose 自動録画（バイナリ・全 bone）はそのまま回す＝完全忠実度のバックアップ

### 手動・外部記録

- scrcpy（片方の HMD ミラー）を OBS で録画 / 室内カメラ任意 / **室内音声**（RQ4 の発話注釈用）
- ファシリテータ観察シート（紙）: 発明されたジェスチャー、質問ゲームの成否、崩壊・笑いの時刻、フェーズ境界時刻

### トラッキングロストの扱い（重要）

マーカー無し条件で手が消えると「無に向かって話す」時間が混入する。**ロスト時は最終姿勢でフリーズ**（フェードで消さない・調査ビルドの既定）し、ロスト区間をログする。分析時に該当区間を除外。

## 5. 分析指標（オフライン）

- RQ2: カード保持中の「カード法線が最も向いている手ランドマーク」の時間分布・人別ヒストグラム
- RQ3: 図指差し結果（主）/ 発話中の頭方向と手ランドマークの角度オフセット分布（副）
- RQ1: 質問ゲーム正答率・到達情報量の定性まとめ・ジェスチャーレパートリーのコーディング
- RQ4: 室内音声の発話注釈（質問形式の割合、Yes/No への収束、発話速度の変化）

## 6. 制約・限界（明記して受容）

- Quest 3 は視線計測不可 → 頭方向プロキシは弱い。図指差し＋カード照準で三角測量
- 同室のため音は筒抜け（手役の笑い等）→ 統制実験ではなく半構造化観察として扱い、漏れはデータ化
- N が小さく、知人ペアか初対面かで RQ4 が大きく変わる → **ペアの知己関係を記録**（できれば固定）
- 箱頭のフルアバター側も「目がどこか」曖昧 → インタビューで「人アバターの目はどこだった？」も聞いて影響を見積もる
