# 手アバター調査 — 実施プロトコル（ファシリテータ用）

設計の背景 → [study-design.md](study-design.md)。**パイロット1ペア実施後に本書を改訂して凍結する。**

## 0. 事前準備

### 機材チェックリスト

- [ ] Quest 3 ×2（充電・ガーディアン設定済み・調査ビルドインストール済み）
- [ ] PC（adb 接続・scrcpy・OBS・このプロトコル）
- [ ] 室内音声レコーダー（スマホ可）・任意で室内カメラ
- [ ] 紙: 観察シート / 手の図（指差し用）×人数分 / 同意書 / 役割インストラクションカード
- [ ] 席: **パーテーション等で区切り**、互いの実身体・実手元・装着動作が見えない配置（音声は筒抜けで良い — 生声が仕様）
- [ ] 同意書 ×2（[consent-template.md](consent-template.md)。**倫理審査の要否を事前確認済みであること**）

### 参加者への事前説明（両者共通）

- VR 酔いしたら即申告・中断可 / HMD は各自専用（交換しない）
- ログ（動作データ）・映像・音声を記録すること、目的は後で説明すること（事後ブリーフィング）に同意を得る
- 練習: 装着 → 手が見えること → ピンチで物が掴めることを各自 1 分

### 起動 runbook（PC から）

```powershell
# 1. ホスト役の Quest（シリアル A）— 例: 人役 (full)
adb -s <serialA> shell am start -n com.UnityTechnologies.com.unity.template.urpblank/com.unity3d.player.UnityPlayerActivity -e tdv_mode host -e tdv_role full

# 2. クライアント役の Quest（シリアル B）— 手役 (hand)
adb -s <serialB> shell am start -n com.UnityTechnologies.com.unity.template.urpblank/com.unity3d.player.UnityPlayerActivity -e tdv_mode client -e tdv_ip <hostQuestのIP> -e tdv_role hand

# 役割交代（HMD はそのまま、role を入れ替えて再起動）
adb -s <serialA> shell am force-stop <pkg> && （上記を role を逆にして再実行）

# フェーズマーク（ホスト宛て）
curl "http://<hostQuestのIP>:7780/mark?label=<任意ラベル>"

# セッション後のログ回収
adb -s <serialA> pull /sdcard/Android/data/<pkg>/files/ ./logs/<日付_ペアID>/
```

近接センサ無効化（装着外で動かす検証時のみ。実セッションでは不要）。

## 1. セッション台本（40分目安）

> 表記: 🗣=ファシリテータの発話例 / 👁=観察ポイント / ⏱=目安時間 / 📌=フェーズマーク送信

### Phase 0: 同期クラップ ⏱1分

両者装着・接続確認後、ファシリテータが **HMD マイクと室内レコーダーに聞こえる位置で1回大きく拍手**（映像にも入れる）。📌 `clap`
→ 全ストリーム（CSV / scrcpy 録画 / 室内音声）の時刻合わせ基準。

### Phase 1: 自由対面 ⏱2–3分 📌 `phase1`

🗣（人役へ）「向かいに『手』がいます。あいさつしてみてください」 — それ以上の指示はしない。
🗣（手役へ・事前カードで）「あなたは喋れません。手の動きだけで応えてください」

👁 人は最初どこを見て話すか / 手はどう「返事」を発明するか / 距離の取り方

### Phase 2: カード課題 ⏱5分 📌 `phase2`

テーブル上に絵カード（片面絵柄）。
🗣「カードを取って、手に見せながら好きかどうか聞いてみてください」

👁 **カードの絵をどこへ向けるか**（自動ログで取れるが、明らかな照準行動はシートにも記録）/ 手の Yes/No 表現

### Phase 3: 質問ゲーム ⏱5分 📌 `phase3`

🗣「手には『好きなもの』が1つあります（手役には事前カードで提示）。声で質問して当ててください。手はジェスチャーだけで答えます」

👁 質問が Yes/No 形式へ収束するか / 指数え・指差しの発生 / 正答までの質問数（**シートに記録**: 質問・応答ジェスチャ・成否）

### Phase 4: 協調配置 ⏱5分 📌 `phase4`

手役だけに見える目標配置（3つの小物の位置）。
🗣「手だけが完成形を知っています。手の指示に従って、テーブルの上を完成させてください」

👁 指差し・ビート・OK/NG サインの発明 / 行き詰まりと打開 / 完成までの時間

（余裕があれば難化バリアント: 空書禁止 or 向き付き配置）

### Phase 5: 役割交代 ⏱2分 📌 `swap`

HMD はそのまま、adb で role を入れ替えて再起動。新しい手役に禁止事項カードを渡す。

### Phase 6: 短縮再実施 ⏱10分 📌 `phase6-1` 等

Phase 1 → 3 → 4 を短縮版で（カード課題は時間があれば）。
👁 **手役経験者は人役になったとき、手への接し方が変わるか**（どこを見るか・話し方）

### Phase 7: 事後インタビュー ⏱8分（個別に）

**図指差し（必須・RQ3 の主計測）**: 手の絵を渡し
🗣「手の『目』はどこにあったと思いますか。絵に丸をつけてください」

続けて両者に:

1. どこに向かって話して（見て）いましたか
2. （手役に）**あなたはどこから相手を見ているつもりでしたか**。自分の「目」はどこでしたか
3. 人アバター（箱頭）の目はどこにあると思いましたか
4. 伝わらなくて困った瞬間は？どう工夫しましたか
5. 手は「誰か」に感じましたか、「道具」に感じましたか
6. （手役に）喋れないことはどれくらいもどかしかったですか

最後に調査目的を説明（事後ブリーフィング）。

## 2. 役割インストラクションカード（印刷用）

### 手役カード

> あなたは「手」です。
> - 喋らない・ささやかない・口で音を出さない（笑ってしまうのは仕方ないです）
> - 伝え方は自由です。手の動きだけで相手とやりとりしてください
> - テーブルの物は親指と人差し指の「つまみ」で掴めます

### 人役カード

> あなたは普通に喋れます。向かいの「手」は喋れませんが、あなたの声は聞こえています。
> 自由に話しかけて、やりとりしてみてください。

## 4. トラブル対応

| 症状 | 対処 |
|---|---|
| 接続が切れた | 両アプリ再起動（v1 仕様）。CSV はファイル分割されるので問題なし |
| 手のトラッキングが頻繁に落ちる | 照明を明るく / 袖をまくる / テーブル面の反射を避ける |
| 酔い・不調 | 即中断。データは破棄してよい |
| ガーディアン警告が出る | その場で立ち位置を調整（席アラインは自動追従） |
